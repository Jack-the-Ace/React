단계1. Design Thinking  
사용자 입장에서 문제를 분석하시오. 

다음 질문에 답하시오. 

① 사용자는 어떤 불편함을 가지고 있는가? 
	- 몇 분 전에 작성된 글인지 한번에 알아보기 불편하다.

② 이 서비스를 사용하면 어떤 점이 편리해지는가? 
	- 현재 날짜 및 시간을 따로 체크하지 않아도 한 순간에 이 글이 얼마나 최신글인지 알아보기 쉬워진다.

③ 사람들이 가장 많이 사용하는 기능은 무엇일까?
	- 댓글 쓰기


단계 2. Logical Thinking 
게시판 기능을 기능 단위로 분해하시오. 
목록 조회, 게시글 보기, 게시글 작성, 게시글 수정, 게시글 삭제 
	- 게시글 목록 조회 (index.html): 전체 게시글 리스트를 화면에 테이블 형태로 출력.
	- 게시글 상세 보기 (view.html): 선택한 특정 게시글의 세부 내용(제목, 작성자, 본문 등)을 표시.
	- 게시글 작성 (write.html): 새로운 게시글의 제목, 작성자, 비밀번호, 내용을 입력받아 저장.
	- 게시글 수정 (edit.html): 기존에 작성된 게시글을 수정하고 반영.  


그리고 React Component 구조를 작성하시오.
	-  App (최상위 컴포넌트)
		- 전체 애플리케이션의 라우터 처리(BrowserRouter) 및 화면 전환 제어 (react-router-dom 활용)
		- Index.jsx (경로: ./pages/)
			- 기존 index.html을 대체. 게시글 목록을 보여주는 화면.
		- View.jsx (경로: ./pages/view/:no)
			- 기존 view.html을 대체. 특정 게시글의 상세 내용을 보여주는 화면.
		- Write.jsx (경로: ./pages/write)
			- 기존 write.html을 대체. 새 글 등록 폼 화면.
		- Edit.jsx (경로: ./pages/edit/:no)
			- 기존 edit.html을 대체. 기존 글 수정 폼 화면.
